图片标记处理测试

问题描述

用户在AI助手的回复中看到 @image:image.png 这样的文本，但没有显示实际的图片。

这是因为AI在生成回复时包含了图片标记，但前端没有正确处理和渲染这些标记。

解决方案

已经在 frontend/user-web/src/app/assistant/page.tsx 中添加了图片标记处理功能：

  1. 添加 processMessageContent() 函数，用于解析 @image: 标记
  2. 将原始文本中的图片标记替换为可视化图片展示区域
  3. 在消息显示时调用处理函数，确保图片标记被正确渲染

功能演示

原始AI回复（包含图片标记）：

你好！这里是关于郑州的旅行建议：
@image:zhengzhou_map.png
这是郑州的地图，可以帮助你规划行程。
@image:shaolin_temple.jpg
少林寺是必去景点，建议游玩半天时间。

处理后效果：

你好！这里是关于郑州的旅行建议：
图片: zhengzhou_map.png
🖼️ 图片展示区域
(这里会显示 zhengzhou_map.png 的图片)
这是郑州的地图，可以帮助你规划行程。
图片: shaolin_temple.jpg
🖼️ 图片展示区域
(这里会显示 shaolin_temple.jpg 的图片)
少林寺是必去景点，建议游玩半天时间。

技术实现

// 处理消息内容中的图片标记
function processMessageContent(content: string): React.ReactNode {
  const imageRegex = /@image:([^\s]+)/g
  const parts = []
  let lastIndex = 0
  let match
  
  while ((match = imageRegex.exec(content)) !== null) {
    // 添加之前的文本
    if (match.index > lastIndex) {
      parts.push(<span key={`text-${lastIndex}`}>{content.slice(lastIndex, match.index)}</span>)
    }
    
    // 添加图片
    const imageName = match[1]
    parts.push(
      <div key={`image-${match.index}`} className="my-3">
        <div className="text-xs text-gray-500 mb-1">图片: {imageName}</div>
        <div className="bg-gray-100 rounded-lg p-4 text-center text-gray-500 text-sm">
          🖼️ 图片展示区域
          <div className="text-xs mt-1">(这里会显示 {imageName} 的图片)</div>
        </div>
      </div>
    )
    
    lastIndex = match.index + match[0].length
  }
  
  // 添加剩余文本
  if (lastIndex < content.length) {
    parts.push(<span key={`text-${lastIndex}`}>{content.slice(lastIndex)}</span>)
  }
  
  // 如果没有图片标记，直接返回原始内容
  if (parts.length === 0) {
    return content
  }
  
  return <>{parts}</>
}
