## Bug 修复：Agent temperature 参数（09:57） 

### 问题 
Moonshot kimi 系列模型只允许 `temperature=1`，但 Agent 循环代码里硬写了 `temperature=0.7`，导致 400 错误。 

### 修复（app.py 第5373行） 
```python 
# 修复前 
response = client.chat.completions.create(..., temperature=0.7, ...) 

# 修复后 
_temp_kwargs = {} if provider == "moonshot" else {"temperature": 0.7} 
response = client.chat.completions.create(..., **_temp_kwargs, ...) 
``` 

### 注意 
Moonshot 模型不传 temperature 参数（默认为1），其他模型（zhipu/openai）继续使用 0.7。 

## Bug 修复：Agent SSE 直连后端（10:05） 

### 问题 
前端 Agent 模式的 SSE fetch 用的是相对路径 `/api/agent/chat`，经过 Next.js rewrite 代理时会缓冲响应，导致 SSE 流无法正常读取，直接抛异常显示"AI 服务暂时繁忙"。 

### 根本原因 
Next.js rewrite/proxy 不支持 SSE 流式透传，会把响应缓冲起来。 

### 修复（assistant/page.tsx） 
```typescript 
// 修复前（经过 Next.js proxy，SSE 被缓冲） 
const res = await fetch('/api/agent/chat', ...) 

// 修复后（直连后端，绕过 Next.js rewrite） 
const backendBase = process.env.NEXT_PUBLIC_API_URL || 'http://127.0.0.1:5001' 
const res = await fetch(`${backendBase}/api/agent/chat`, ...) 
``` 

### 规律 
**所有 SSE/流式请求必须直连后端**，不能走 Next.js rewrite，否则流会被缓冲。
