   - 优化 `onSendAgent()` 中的 `setMsgs` 调用，一次性添加用户和AI消息
   
2. 修改前:
   ```typescript
   const nextMsgs = [...msgs, userMsg]
   setMsgs(nextMsgs)  // 第一次更新
   const msgsWithAssistant = [...nextMsgs, assistantMsg]
   setMsgs(msgsWithAssistant)  // 第二次更新（可能导致重复）
   ```
   
   修改后:
   ```typescript
   const msgsWithAssistant = [...msgs, userMsg, assistantMsg]
   setMsgs(msgsWithAssistant)  // 一次性更新
   ```

### NearbyMap 无限加载修复（2026-04-10 晚）

#### 问题
组件无限加载，一直显示"正在搜索周边..."

#### 原因
1. `center` 是数组 `[number, number]`，每次组件重新渲染，即使值相同，数组引用也会改变，导致 useEffect 被反复触发
2. 没有重试次数限制，失败后会无限重试

#### 修复内容
1. 使用 `useMemo` 稳定 `center` 的引用：
   ```typescript
   const center = useMemo(() => {
     const [lng, lat] = propCenter
     if (lng === 0 || lat === 0 || isNaN(lng) || isNaN(lat)) {
       return null
     }
     return [lng, lat] as [number, number]
   }, [propCenter[0], propCenter[1]])
   ```

2. 添加重试次数限制（最多 3 次）：
   ```typescript
   const retryCountRef = useRef(0)
   const MAX_RETRY = 3
   ```

3. 修改地图初始化 useEffect 依赖为字符串格式，避免引用问题：
   ```typescript
   }, [center ? `${center[0]},${center[1]}` : '', destinationName])
   ```

- `frontend/user-web/src/components/NearbyMap.tsx`
