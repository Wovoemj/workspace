## 401未授权错误修复与用户指导优化

### 问题
用户输入"河南3天"后，前端控制台显示：`api/itinerary/generate:1 Failed to load resource: the server responded with a status of 401 (UNAUTHORIZED)`

### 根本原因分析
1. **token存在但无效**：localStorage中有`auth_token`，但token已过期或格式错误
2. **后端认证失败**：JWT token验证失败，返回401错误
3. **前端错误处理不够友好**：只显示"无效的token"，没有提供解决方案

### 后端认证流程
```python
@app.route('/api/itinerary/generate', methods=['POST'])
@rate_limit('itinerary_generate', limit=10)
def generate_itinerary():
    # 1. 检查Authorization header
    auth_header = request.headers.get('Authorization', '')
    if not auth_header.startswith('Bearer '):
        return jsonify({'success': False, 'error': '未登录，请先登录'}), 401
    
    # 2. 提取并验证token
    token = auth_header[7:]
    try:
        payload = jwt.decode(token, get_secret_key(), algorithms=['HS256'])
        user_id = payload.get('user_id')
        if not user_id:
            return jsonify({'success': False, 'error': '无效的token'}), 401
    except jwt.ExpiredSignatureError:
        return jsonify({'success': False, 'error': 'token已过期，请重新登录'}), 401
    except jwt.InvalidTokenError:
        return jsonify({'success': False, 'error': '无效的token'}), 401
    
    # 3. 检查用户是否存在
    user = User.query.get(user_id)
    if not user:
        return jsonify({'success': False, 'error': '用户不存在'}), 401
```
