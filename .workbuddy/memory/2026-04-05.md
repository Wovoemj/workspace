# 2026-04-05 

## 问题诊断 
1. 后台仪表盘显示 HTTP 401 错误 
2. `/api/stats` 接口使用了 `@admin_required` 装饰器，需要管理员认证 
3. 前端调用 `/api/stats` 时没有携带认证头 

## 解决方案 
- 启动完整服务栈 (backend + user-web) 
- 检查认证机制 
- 确保管理员登录状态正确传递 

## 修复：管理页面 HTTP 401 认证问题 
- **根因**：前端管理页面使用裸 `fetch` 调用后端管理员接口（`/api/stats`、`/api/admin/*`），没有携带认证头 
- **修复**： 
  - `apiClient.ts` 新增 `adminAuth` 选项，从 `localStorage.admin_token` 读取管理员 token 
  - `admin/page.tsx` 改用 `apiClient` 并传入 `adminAuth: true` 
  - `admin/users/page.tsx` 同样修复，从使用 `auth_token` 改为 `adminAuth: true` 
- **修改文件**：`frontend/user-web/src/lib/apiClient.ts`、`frontend/user-web/src/app/admin/page.tsx`、`frontend/user-web/src/app/admin/users/page.tsx` 

## 新功能：管理员登录密码查看按钮 
- `AdminGuard.tsx` 密码输入框添加 👁 眼睛图标按钮，点击切换密码显示/隐藏 
- 使用 lucide-react 的 `Eye`/`EyeOff` 图标 
- **修改文件**：`frontend/user-web/src/components/AdminGuard.tsx`
